# 终端滚动条问题-交接说明

日期：2026-03-03

## 问题描述
- 在自定义分屏/重排（reflow）场景下，终端滚动条经常不在最新输出位置。
- 表现为：看起来“停在历史位置”，不是跟着最新输出。

## 已确认范围
- 只需要修改 **WAVE 仓库**（`E:/code/waveterm-main`）。
- 不需要修改 `E:/code/cx`。

## 已做修复（代码位置）
- 文件：`frontend/app/view/term/termwrap.ts`
- 关键点：
  - 保存滚动状态由单一 `viewportY` 改为 `{ viewportY, wasAtBottom }`
  - 如果重排前在底部，重排后强制回到底部（包含一次 `requestAnimationFrame` 兜底）
  - 如果重排前不在底部，保持用户阅读位置
  - 重排流程在 `finally` 清理缓存滚动状态，避免后续污染

## 验证记录
- `npm test -- --run`：通过
- `npm run dev:fresh`：可启动并进入开发窗口

## 给“另一个窗口 AI”的一句话
- 请先读取本文件，再查看 `frontend/app/view/term/termwrap.ts` 的滚动恢复逻辑，不要去改 `cx` 仓库。

## 语音播报目标定义（重点）
- 语音**只播报**“AI 最终正式回复正文”，也就是终端里你看到的那块 **AI 回复背景色区域**（assistant 段）。
- 在 Codex 终端格式里，默认对应“最新一段 `• ` 开头的 assistant 回复段 + 其连续正文行”。
- 结束边界：遇到下一条用户提示符（如 `› `）或下一段非正文边界；严格模式下需确认回复已闭合（出现后续提示符）才播报。

### 明确不播报内容
- 工具调用/工具结果：`Called ...`、`Calling ...`、工具参数/JSON 输出。
- 运行状态与进度：`Working ...`、`esc to interrupt`、MCP 服务器启动/停止状态。
- 底栏/遥测：模型名、剩余额度、`Inference/Streams` 统计、快捷键提示。
- 中断与错误提示：`Conversation interrupted`、`Something went wrong`、`/feedback` 提示。
- shell 提示符与系统噪声：`PS ...>`、`user@host:...$`、Node deprecation 等。

### 失败兜底
- 若当前屏幕没有“已完成的 AI 正式回复段”，返回：`没有检测到可播报的 AI 正式回复。`

## 给另一个窗口 AI 的可复制指令
- 只把“AI 回复背景色那一整段正文”当成 TTS 输入；不要读工具调用、状态行、底栏、提示符或错误提示。若最新回复尚未闭合（还在生成），先不播报，直到形成完整 assistant 段再播报。
