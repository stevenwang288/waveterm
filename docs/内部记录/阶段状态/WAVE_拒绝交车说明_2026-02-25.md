# WAVE 拒绝交车说明（2026-02-25）

> 结论：本轮不交车。原因不是“你起错脚本”，而是**版本一致性 + 本地 TTS 引擎定义不清**导致“看起来同一套代码、实际跑的不是同一个东西”，并且会稳定触发你反馈的几类问题（播报乱读、TTS 质量差、端口冲突、启动弹浏览器等）。

## 1. 交车门槛（本轮未达到）

### A. 版本一致性（必须可证明“你装的=我打包的”）
- 需要能在用户侧明确确认：安装包对应的 `wavesrv` BuildTime/Commit，与仓库构建一致。
- 当前未做到：同版本号下存在不同 BuildTime 的安装态二进制，导致用户“以为装了新包，实际还在跑旧包”。

### B. 语音链路一致性（必须稳定复现同一套 TTS 行为）
- “本地/Edge TTS”需要明确是：
  1) 你自己运行的兼容服务（例如 `127.0.0.1:5050/v1/audio/speech`），还是
  2) Wave 内置提供的本地服务（如果内置服务音质不达标，则不得默认启用）。
- 当前未做到：端口被 Wave 自身或旧版进程占用时，会出现**低质量 Windows 兜底**或**被错误进程接走请求**，最终表现为“怎么改设置都像没生效”。

### C. 自动播报（必须只播报“该播报的”）
- 自动播报应只读“最终回复”，不得把状态噪声（如 reconnect/502、进度提示、历史滚动）当成要播报的正文。
- 当前仍会出现：把错误提示/状态信息也当正文播报，用户体验不可接受。

## 2. 已确认的事实证据（可复现）

### A. 你机器上同时存在“已安装旧版”和“当前仓库构建”
- 已安装程序内的 `wavesrv` BuildTime：`202602222318`（旧）
- 仓库 `dist/bin/wavesrv.x64.exe` BuildTime：`202602250357`（新）
- 结论：就算 UI 显示同样的版本号，**后端二进制并不一致**，行为会完全不同。

### B. 端口冲突会直接导致“听到垃圾 Windows TTS”
- `127.0.0.1:5050/5051` 在用户环境中可能被“已安装旧版 Wave”占用。
- 当你在设置里选“本地/Edge TTS”且 Endpoint 留空时，前端会默认请求 `5050`。
- 如果此时 `5050` 被旧版占用，**请求会被旧版接走**，导致你听到“Windows 自带兜底”的低质量声音（并且看起来像“设置没生效”）。

### C. 当前代码的“内置本地 TTS”实现确实是 Windows 兜底
- 内置服务通过 PowerShell + `System.Speech` 合成音频。
- 这条链路无法保证与 UI 中的 `zh-CN-xxxNeural` 音色一致（多数情况下会回退到系统已安装的普通语音），因此会被主观感受为“机器人音”。

## 3. 用户现场反馈（本轮高优先级未满足）
- 自动播报会“什么都读”（包含状态/噪声/历史）。
- TTS 模型/音色经常“看起来设置对了，听起来还是错的”。
- 启动时会弹出浏览器并卡在选择用户（不应发生或必须是显式用户操作触发）。
- 任务栏/图标状态不稳定（启动后找不到正确实例）。
- 打包安装后“完全不是同一个东西”（核心是版本一致性不可验证）。

## 4. 已做的修复尝试（但因上述根因，仍不足以交车）
- 增加 dev 运行隔离脚本：每次启动使用独立 config/data，避免“脏配置”带来错觉。
- 打包前校验输出：避免把陈旧的 `dist/*` 打进安装包。
- 限制自动打开外部浏览器：仅在明确 user gesture 时允许 `openExternal/window.open`。
- 内置本地 TTS 改为“默认不启动/需显式开启”（目的：不再抢占 5050/5051，不再强制兜底）。

> 上述改动如果没有被实际安装进用户运行的版本（或旧版仍在后台占用端口），用户侧体验会**完全看不到改善**。

## 5. 未解决项（阻塞交车）

### A. “本地 Edge TTS”定义缺失
需要做出明确决策并实现：
1) **本地=外部兼容服务**（Wave 不内置低质量兜底；Endpoint 必填/或仅在检测到端口已有服务时才使用默认端口），或
2) **本地=Wave 内置高质量引擎**（必须达到用户可接受的音质；否则不允许默认启用）。

### B. 交付标识缺失（导致“你认可的版本”无法锁定）
用户反馈“某次打包版本非常完美”，但缺少：
- 安装包命名中带 BuildTime/Commit
- 应用内可一眼看到的 BuildTime/Commit（About/设置页）
导致后续无法证明“回滚到你认可的那一版”。

## 6. 建议的下一步（重新立项的交车路径）
- 1) 先补齐“版本可验证”：安装包/应用内展示 BuildTime + Commit；并强制安装前关闭旧进程。
- 2) 明确“本地 TTS 策略”：要不要内置、内置用什么、是否允许 Windows 兜底；不允许再出现“UI 叫 Edge，实际是 Windows”这种误导。
- 3) 把自动播报的过滤规则写成验收标准 + 自动化 smoke（确保不会播报 reconnect/502/进度噪声）。
